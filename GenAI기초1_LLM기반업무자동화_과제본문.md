# GenAI 기초 1: LLM 기반 업무 자동화

## 1. 미션 소개

> "ChatGPT한테 물어봤는데, 생각보다 별로던데요?"

한 번쯤 이런 허탈함을 느껴본 적이 있을 겁니다.  
분명 세상을 바꿀 혁명이라고 하는데, 막상 내 업무를 맡기면 엉뚱한 대답만 늘어놓거나 겉도는 말만 반복하는 AI.  
그런데 옆자리 동료는 같은 AI로 보고서 초안부터 데이터 요약까지 뚝딱 해치웁니다.  
도대체 뭐가 다른 걸까요?

답은 의외로 간단합니다.

**질문을 어떻게 설계했느냐, 그 차이입니다.**

같은 모델이라도 프롬프트 한 줄이 달라지면 결과물의 수준은 완전히 바뀝니다.

`그냥 물어보는 것`과 `제대로 시키는 것` 사이의 간극을 직접 체감하는 것이 이번 미션의 출발점입니다.

이번 미션에서는 대규모 언어 모델, 즉 **LLM(Large Language Model)** 의 작동 원리를 이해하고, 프롬프트 엔지니어링을 통해 원하는 텍스트 결과물을 정교하게 제어하는 기술을 익힙니다.

먼저 **3종 이상의 LLM**을 동일한 업무 과업에 투입해 성능을 비교하고, "느낌"이 아닌 평가 축과 근거로 최적의 모델을 선정하는 방법을 학습합니다.

이어서 특정 페르소나를 가진 AI 챗봇의 시스템 프롬프트를 직접 설계하고, Few-shot 예시와 단계적 추론 유도 기법을 적용해 프롬프트를 버전별로 개선해 나갑니다.

단순한 질문-답변을 넘어 **10턴 이상의 대화**에서 문맥을 유지하며 논리적으로 대화를 이끌어가는 텍스트 생성 파이프라인을 구축하고, 환각(Hallucination)을 체계적으로 검증하는 과정까지 경험합니다.

이 과정을 마치면, AI를 단순한 검색 대용이 아니라 업무 생산성을 끌어올리는 핵심 파트너로 다룰 수 있게 됩니다.

## 2. 최종 결과물

다음 3가지 산출물을 포함한 **업무 자동화 프롬프트 패키지 1개**를 완성한다.

### 2.1 LLM 모델 비교·선정 보고서

동일한 업무 과업 프롬프트를 3종 이상의 모델에 입력하고, 모델별 결과 비교표와 최종 선정 결론을 정리한 문서

포함 내용:

- 비교 대상 모델명
- 사용 환경
- 동일 입력 프롬프트
- 평가 축과 점수
- 모델별 결과 비교
- 최종 선정 근거

---

### 2.2 시스템 설계 문서

타겟 사용자, 업무 문제, 페르소나, 시스템/유저 프롬프트, Few-shot 예시, 검증 전략을 포함한다.

포함 내용:

- 문제 정의
- 타겟 사용자
- 입력 데이터 형태
- 출력 규격
- 입력 템플릿
- 페르소나 정의
- 시스템 프롬프트
- Few-shot 예시
- 환각 검증 전략
- 프롬프트 개선 이력
- 최종 프롬프트 전문

---

### 2.3 실행 로그

선택한 업무 과업 시나리오로 **10턴 이상 실제 대화**를 수행한 전문과, 문제 발생 지점 및 수정 결과 요약을 담은 로그

포함 내용:

- 시나리오 설명
- 10턴 이상 대화 전문
- 환각 또는 문맥 끊김 발생 여부
- 조건 변경 반영 여부
- 수정 결과 요약

---

## 3. 과제 목표

이 과제를 마친 후, 학습자는 아래 내용을 스스로 설명할 수 있어야 한다.

1. LLM이 **확률적 생성** 기반으로 동작하며, 왜 환각(Hallucination)이 발생하는지 설명할 수 있다.
2. Zero-shot, Few-shot, 단계적 추론 유도 기법의 차이를 구분하고 업무 과업에 맞게 적용할 수 있다.
3. 3종 이상의 LLM을 비교할 때, 느낌이 아니라 평가 축과 근거를 통해 선택 이유를 설명할 수 있다.
4. 페르소나와 시스템 프롬프트가 결과 품질에 미치는 영향을 설명할 수 있다.
5. 생성 결과를 테스트로 검증하고, 프롬프트를 반복 개선하는 과정을 재현 가능하게 설명할 수 있다.

## 4. 기능 요구 사항

### 4.1 모델 비교 및 선정

서로 다른 3개 이상의 LLM을 선정해 동일한 업무 과업을 수행시킨다.

예시 모델:

- GPT-4o
- Claude 3.5 Sonnet
- Gemini 1.5 Pro

또는 동급/유사 목적의 최신 모델을 사용할 수 있다.

모델 비교 보고서에는 아래 항목을 반드시 포함한다.

| 필수 항목 | 설명 |
|---|---|
| 비교 대상 모델명 | 사용한 LLM 모델명 |
| 사용 환경 | 웹, 앱, API 중 무엇인지 기록 |
| 평가 축 | 최소 4개 이상 |
| 점수 | 모델별 1~5점 |
| 근거 | 점수 부여 이유 |
| 최종 선정 결론 | 왜 이 모델이 업무 과업에 최적인지 3줄 이상 설명 |

---

### 4.2 업무 자동화 과업 1개 선택 및 문제 정의

아래 예시 중 1개 이상을 선택하거나, 본인 업무에 준하는 과업을 정의한다.

예시 과업:

- 메일 초안 작성
- 회의록 요약
- 보고서 개요 생성
- 요구사항 정리
- 업무 프로세스 분석
- 시스템 산출물 초안 생성

선택한 과업에 대해 다음을 문서에 명시한다.

| 항목 | 설명 |
|---|---|
| 타겟 사용자 | 누가 사용하는가 |
| 입력 데이터 형태 | 어떤 정보를 받는가 |
| 출력 규격 | 어떤 형식으로 내야 하는가 |
| 입력 템플릿 | 재사용 가능한 입력 양식 |

입력 템플릿 예시:

```text
[업무 과업]
[목적]
[대상 사용자]
[입력 자료]
[핵심 포인트]
[출력 형식]
[톤]
[금지 사항]
[필수 포함 항목]
