[과제] GenAI 기초 1: LLM 기반 업무 자동화
1. 미션 소개
"ChatGPT한테 물어봤는데, 생각보다 별로던데요?"
이러한 허탈함을 극복하기 위해, 본 미션에서는 프롬프트 엔지니어링을 통해 AI를 단순 검색 도구가 아닌 업무 생산성을 높이는 핵심 파트너로 만드는 과정을 학습합니다.

핵심 학습 내용:
3종 이상의 LLM 성능 비교 및 선정 (평가 축과 근거 기반)
페르소나 기반 시스템 프롬프트 설계
Few-shot 예시 및 단계적 추론 유도(Reasoning Prompt) 적용
10턴 이상의 문맥 유지 및 환각(Hallucination) 검증
2. 최종 결과물
다음 3가지 산출물을 포함한 업무 자동화 프롬프트 패키지를 완성합니다.

LLM 모델 비교·선정 보고서: 3종 이상의 모델 비교표 및 선정 근거.
시스템 설계 문서: 페르소나, 프롬프트(v1→v2 개선), Few-shot, 검증 전략.
실행 로그: 10턴 이상의 대화 전문 및 문제 해결 과정 요약.
3. 과제 목표
LLM의 확률적 생성 원리와 환각 발생 이유 설명 가능.
Zero/Few-shot, 단계적 추론 유도 기법의 구분 및 적용.
평가 축을 통한 객관적인 모델 선정.
페르소나와 시스템 프롬프트가 결과 품질에 미치는 영향 이해.
프롬프트 반복 개선 및 사실 기반 검증 과정 재현.
4. 기능 요구 사항
모델 비교 및 선정 (최소 3종)
비교 대상 모델명, 사용 환경(웹/API 등) 명시.
평가 축 최소 4개(정확성, 한국어, 문맥 유지, 형식 준수, 속도, 비용 등) 선정 및 점수(1~5점) 부여.
최종 선정 결론: 왜 이 모델이 최적인지 3줄 이상의 근거 제시.
업무 자동화 과업 및 입력 템플릿
과업 예시: 메일 초안, 회의록 요약, 보고서 개요, 요구사항 정리 등.
입력 템플릿 필수 항목: [목적], [대상], [핵심 포인트], [톤], [금지어], [필수 포함 항목].
페르소나 및 시스템 프롬프트
페르소나 정의: 이름, 역할, 전문 분야, 말투, 우선순위 등.
시스템 프롬프트 필수 요소: 역할과 목표, 출력 형식 규칙, 안전장치(모르면 모른다고 하기), 사실 확인 규칙.
Few-shot 및 단계적 추론
Few-shot: 좋은 입력→좋은 출력 예시 3개 (1개는 모호한 입력 포함).
단계적 추론(v1 vs v2): v1(단순 지시)과 v2(단계적 접근) 비교. 최종 답변에는 핵심 근거 3개만 제시하도록 규칙 설정.
환각(Hallucination) 검증
사실/수치/정책 기반 질문 5개 이상 수행.
Pass 기준: 정답 제시 또는 근거 부족 시 확인 절차 제안.
Fail 기준: 근거 없는 단정적 답변, 전제 불명확 시 임의 생성.
10턴 대화 로그 및 문맥 유지
조건 변경(톤, 대상, 정책 변경) 상황을 포함하여 10턴 이상 수행.
이전 요구사항을 일관되게 유지하는지 확인.
5. 결과 예시 (참고용)
모델 비교 축: 정확성, 한국어 자연스러움, 긴 문맥 유지, 형식 준수, 속도, 비용, 도구 연동.
입력 템플릿 예시:
[업무 과업] 회의록 요약
[원문] (회의 메모) / [원하는 결과] 결정사항, Action Items, 리스크, 후속 일정
단계적 추론 프롬프트(v2):
목표 출력 항목 선언.
정보 부족 시 최대 3개 확인 질문.
템플릿으로 출력.
최종 답변에 핵심 근거 3개만 요약.
6. 제약 사항
재현성: 모델명, 요금제, 사용 채널, 날짜, 주요 설정(Temperature 등) 기록.
안전/윤리: 개인정보 및 폭력적/선정적 콘텐츠 생성 금지.
편집: 대화 로그는 가독성을 위해 정리하되, 원본 로그는 별도 보관.
7. 보너스 과제 (선택)
보너스 1: 커스텀 인스트럭션/GPTs를 활용한 봇 배포.
보너스 2: 이미지 생성 AI를 활용한 결과물의 시각화.
