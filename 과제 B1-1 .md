# WBS1. 업무 과업 정의 최종본

## 1. 과업명

**COREERP 계획수립 Help System 프롬프트 패키지**  
**- MPS, MRP II, 자재발주 Q&A 및 문제 대응 자동화 -**

---

## 2. 과업 목적

본 과업은 COREERP의 계획수립 단계에서 사용자가 **MPS, MRP II, 자재발주**와 관련된 업무 절차, 기준정보, 처리 기준, 계획 결과 및 문제 상황을 정확히 이해할 수 있도록 지원하는 **LLM 기반 Help System 프롬프트 패키지**를 설계하는 것이다.

이 Help System은 사용자가 계획수립 과정에서 다음과 같은 상황을 만났을 때 도움을 제공하는 것을 목적으로 한다.

- 계획이 생성되지 않는 경우
- 계획량이 예상과 다른 경우
- 납기 또는 소요일자가 달라지는 경우
- 자재발주 결과가 생성되지 않는 경우
- 기준정보 설정이 계획 결과에 어떤 영향을 주는지 알고 싶은 경우
- 유지보수 또는 컨설팅 관점에서 처리 로직과 원인을 추적해야 하는 경우

---

## 3. 업무 범위

본 과업의 대상 범위는 COREERP의 **계획수립 업무**이며, 세부 범위는 다음과 같다.

| 구분 | 포함 내용 |
|---|---|
| MPS | 주생산계획 수립 및 계획 대상 품목의 생산계획 생성 |
| MRP II | 생산계획, 소요자재계획, 소요재공품계획 수립 |
| 자재발주 | MRP II 결과를 기반으로 원재료 구매계획을 수립하고 주문을 확정하는 단계 |
| 기준정보 영향 분석 | 품목, 조달구분, 조달처, 작업장, BOM, 라우팅, 리드타임, 발주정책 등이 계획 결과에 미치는 영향 분석 |
| 문제 상황 대응 | 계획 미생성, 계획량 차이, 납기 변화, 자재발주 미생성 등 문제 원인 추적 및 대응 방안 제시 |

---

## 4. COREERP 계획수립 프로세스 전제

COREERP에서는 계획수립 프로세스가 다음과 같이 구분되어 설계되어 있다.

```text
MPS
→ MRP II
   - 생산계획 수립
   - 소요자재계획 수립
   - 소요재공품계획 수립
→ 자재발주
   - 원재료 구매계획 수립
   - 주문확정
